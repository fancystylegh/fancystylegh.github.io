CSS选择器！

一. 基本选择器

  1. 通用选择器，会将所有元素匹配并设置样式。
    *{
    border: 1px solid red;
    }
  2. 元素选择器
    p{
    color: red;
    }
  3. id选择器
    #abc{
    font-size: 12px;
    }
  4. 类选择器
    .abc{
    border: 1px solid red;
    }
    也可以使用”元素.class”的形式，限定某种类型的元素。
  5. 属性选择器,中括号里是属性名
    [href]{
    color: red;
    }

[type=”password”]{
border: 1px solid red;
}

[href^=”http”]{
color: red;
}

[href$=”.com”]{
color: red;
}

[href*=”baidu”]{
color: red;
}

[class~=”abc”]{
font-size: 12px;
}

HTML5

[lang|=”en”]{
color: red;
}

二.复合选择器–将不同的选择器进行组合形成新的特定匹配

  1. 分组选择器
    p,span,h1{
    color: red;
    }
    将多个选择器通过逗号分隔，同时设置一组样式。

  2. 后代选择器
    span p{
    color: red;
    }

  3. 子选择器 只能向下一级
    ul > li{
    border: 1px solid red;
    }

  4. 相邻兄弟选择器 匹配和第一个元素相邻的第二个元素
    p + b{
    color: red;
    }

  5. 普通兄弟选择器 匹配和第一个元素后面的所有的元素
    p ~ b{
    color: red;
    }

三. 伪元素选择器
伪选择器分为两种：第一种是下节伪类选择器（:），第二种是伪元素选择器(::)

  1. ::first-line块级首行
    ::first-line{
    color: red;
    }

  2. ::first-letter块级首字母
    ::first-letter{
    color: red;
    }

  3. ::before文本前插入
    a::before{
    content: “点击-“;
    }

  4. ::after 文本后插入
    a::after{
    content: “-请进”
    }

四. 结构性伪类选择器
结构性伪类选择器能够根据元素在文档中的位置选择元素。这类元素都有一个前缀（:）

  1. 根元素选择器,基本不用，总是返回元素
    :root{
    border: 1px solid red;
    }

  2. 子元素选择器,
    ul > li:first-child{
    color: red;
    }
    选择第一个li

ul > li:last-child{
color: red;
}
选择最后一个li

ul > li:only-child{
color: red;
}
选择只有一个子元素的那个子元素

div > p:only-of-type{
color: red;
}
选择只有一个指定类型的子元素的那个子元素

  1. :nth-child(n)系列
    ul > li:nth-child(2){
    color: red;
    }
    选择子元素的第二个元素

ul > li:nth-last-child(2){
color: red;
}
选择子元素的倒数第二个元素

div > p:nth-of-type(2){
color: red;
}
选择特定子元素的第二个元素

div > p:nth-last-of-type(2){
color: red;
}
选择特定子元素的倒数第二个元素

五. UI伪类选择器
UI伪类选择器是根据元素的状态匹配元素

  1. :enabled{
    border: 1px solid red;
    }
    选择启用状态的元素

  2. :disabled{
    border: 1px solid red;
    }
    选择禁用状态的元素

  3. :checked{
    display: none;
    }
    选择勾选的input元素

  4. :default{
    display: none;
    }
    从一组类似的元素中选择默认元素。比如input被勾选的就是默认的。

  5. :valid和:invalid

input:valid{
border: 1px solid red;
}
input:invalid{
border: 1px solid green;
}
输入验证合法与不合法显示时选择的元素

  1. :required和:optional

input:required{
border: 1px solid red;
}
input:optional{
border: 1px solid green;
}
是否具有required属性选择元素

六. 动态伪类选择器–根据条件的改变匹配元素

  1. :link和:visited
    a:link{
    color: red;
    }
    a:visited{
    color: orange;
    }

  2. a:hover{
    color: blue;
    }

  3. a:active{
    color: green;
    }

4.input:focus{
border: 1px solid red;
}
表示获得焦点时

七. 其他伪类选择器

  1. a:not([href*=”baidu”]){
    color: red;
    }
    否定选择器，反选

  2. :empty{
    display: none;
    }
    匹配没有任何内容的元素

  3. :lang(en){
    color: red;
    }
    选择包含lang属性，属性值前缀为en的元素。和属性选择器匹配结果一致。

  4. :target{
    color: red;
    }
    定位到锚点时，选择此元素

  5. ::selection{
    color: red;
    }
    这是一个伪元素选择器，当选择文字时触发选择。

不同的浏览器的标签默认的外补丁和内补丁不同

问题的症状：随便写几个标签，在不加样式控制的情况下，各自的margin和padding差异较大

解决方案：CSS里加*{margin:0;padding:0;}

块属性标签float后，又有横行的margin情况下，在IE6显示的margin比设置的大。（双边距bug）

问题症状：常见症状是IE6中后面的一块被顶到下一行

解决方案：在float的标签样式控制中加入display:inline;将其转化为行内属性。

行内属性标签，设置display:block后采用float布局，又有横行的margin的情况（IE6间距bug）

问题症状：IE6里的间距比超过设置的间距

解决方案：在display:block;后面加入diaplay:inline;display:table;

IE6中3px问题及解决方法

问题症状：当元素使用float浮动后，元素和相邻的元素之间会产生3px的间隙。诡异的是如果右侧的容器没有设置高度时3px的间隙在相邻容器的内部，当设置了高度后又跑到容器的相反侧了。

解决方案：需要使布局在同一行的元素都加上float浮动。

IE6下空元素的高度bug

问题描述：如果一个元素中没有任何内容，当在样式中为这个元素设置了0-19px之间的高度时。此元素的高度始终为19px.

解决方法：

1. 在元素的css中加入：overflow:hidden;

2. 在元素中插入html注释：<!->

3. 在元素中插入html空白符：&nbsp;

4. 在元素的css中加入：font-size:0;

重复文字的bug

问题描述：在某些比较复杂的排版中，有时候浮动元素的最后一些字符会出现在clear清除元素的下面

解决方法：

1. 确保元素都带有display:inline;

2. 在最后一个元素上使用"margin-right:-3px;"

3. 为浮动元素的最后一个条目加上条件注释，<!-[if !IE]>xxx<![endif]->

4. 在容器的最后元素使用一个空白的div，为这个div指定不超过容器的宽度

IE6中z-index失效

问题症状：元素的父级元素设置的z-index为1，那么其子级元素再设置z-index时会失效，其层级会继承父级元素的设置，造成某些层级上调整上的Bug

解决方法：

1. position:relative；改为position:absolute;

2. 去除浮动

3. 浮动元素添加position属性（如relative,absolute等）

不过浏览器兼容也要考虑目标用户，有些使用老旧的浏览器的可能并不是我们的目标客户。
例如：IE8以下的就不兼容了，提示更新浏览器

javaScript事件模型

  • 原始事件模型：ele.onclick = function(){}这种类型的事件模型

  • 冒泡事件模型：事件从事件的发生地（目标元素），一直向上传递，直到document

  • 捕获型事件模型：事件从document向下传递，直到事件的发生地（目标元素）

事件委托机制

事件委托机制指的是：不在事件发生地设立监听函数，而是在事件发生地的父元素或者祖先元素设置监听函数，这样可以大大提高性能。
例如：

要给li元素绑定click元素，使用事件委托机制的话，就只需要给ul绑定click事件。不用给每个li绑定click事件。减少内存的占用，提高效率。

TCP传输的三次握手和四次挥手策略

三次握手：
一个TCP连接需要“三次握手”：

第一次：发送端首先发送一个带SYN标志的数据包给对方。

第二次：接收端收到后，回传一个带有SYN/ACK标志的数据包来传达确认消息。

第三次：发送端再回传一个带ACK标志的数据包，代表握手结束。

四次挥手：
断开一个TCP连接需要“四次挥手”：

第一次：主动关闭方发送一个FIN，用来关闭主动方到被动关闭方的数据传送

第二次：被动关闭方收到FIN包后，发送一个ACK给对方，确认序号为收到序号+1

第三次：被动关闭方发送一个FIN，用来关闭被动关闭方到主动关闭方的数据传送

第四次：主动关闭方收到FIN包后，发送一个ACK给被动关闭方，确认序号为收到序号+1。

cookie

cookie虽然在持久保存客户端数据中提供了方便，分担了服务器存储的负担，但是仍有很多局限性。

优点：极高的扩展性和可用性

1. 通过良好的编程，控制保存在cookie中的session对象的大小

2. 通过加密和安全传输技术（SSL），减少了cookie被破解的可能性

3. 只在cookie中存放不敏感数据，即使被盗也不会有重大的损失

4. 控制cookie的生命周期，使它不会永远有效，即使被窃取也是一个过期的cookie

缺点：

1. cookie的数量和长度有限制，最多有20条，而且每条的长度不能超过4KB

2. 安全性问题

3. 有些状态不能保存在客户端

JS继承的概念

js里通常有两种继承方式：

* 原型链继承（对象间的继承）
* 类式继承（构造函数间的继承）

由于js不像java那样是真正的面向对象的语言，js是基于对象的，它没有类的概念。所以，要想实现继承，可以用js的原型prototype机制或者用apply和call方法去实现

在面向对象的语言中，我们使用类来创建一个自定义对象。然而js中所有的事物都是对象，这就需要用js的原型来创建自定义对象。

我们可以简单的把prototype看作是一个模板，新创建的自定义对象都是这个模板的一个链接。js可以通过构造函数和原型的方式模拟实现类的功能。另外，js类式继承的实现也是依靠原型链来实现的。

position的值，relative和absolute分别是相对于谁定位的？

  • static: 静态的，默认的，没有定位，元素出现在正常流中。
  • absolute: 生成绝对定位的元素，相对于最近一级的定位不是static的父元素进行定位。默认宽度为内容宽度，脱离文档流。
  • fixed:老的IE浏览器不支持，生成绝对定位的元素，相对于浏览器窗口或frame进行定位。默认宽度是内容宽度，脱离了文档流。（通常用于固定顶栏或底栏，遮罩）
  • relative: 生成相对定位的元素，相对于其在普通流的位置进行定位。仍在文档流中，参照物是其本身。使用场景：绝对定位的参照物。
