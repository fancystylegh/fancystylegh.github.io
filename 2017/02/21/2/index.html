cookie

cookie虽然在持久保存客户端数据中提供了方便，分担了服务器存储的负担，但是仍有很多局限性。

优点：极高的扩展性和可用性

1. 通过良好的编程，控制保存在cookie中的session对象的大小

2. 通过加密和安全传输技术（SSL），减少了cookie被破解的可能性

3. 只在cookie中存放不敏感数据，即使被盗也不会有重大的损失

4. 控制cookie的生命周期，使它不会永远有效，即使被窃取也是一个过期的cookie

缺点：

1. cookie的数量和长度有限制，最多有20条，而且每条的长度不能超过4KB

2. 安全性问题

3. 有些状态不能保存在客户端

浏览器的本地存储

在较高版本的浏览器中，js提供了sessionStorage和globalStorage。在HTML5中提供了localStorage来取代globalStorage。

sessionStorage用于本地存储一个session中的数据，这些数据只有在同一个会话页面才能访问并且当前的会话结束后数据也随之销毁。它不是一个持久化的存储，仅仅是会话级别的存储。
而localStorage用于持久化的本地存储，除非主动删除数据，否则数据永远不会过期。

web storage和cookie的区别

web storage是为了更大容量存储设计的，它拥有setItem,getItem,removeItem,clear等方法。web storage仅仅是为了本地存储数据而存在。

cookie的大小是有限的，并且每次请求一个新的页面的时候，cookie都会被发送过去，这浪费了带宽。另外cookie还需要指定作用域，不可以跨域调用。cookie前端开发者自己封装setCookie,getCookie。cookie的作用是与服务器进行交互，作为HTTP规范的一部分而存在。

cookie和session的区别

1. cookie数据存放在客户的浏览器上，session的数据存放在服务器上

2. cookie不安全，别人可以分析存放在本地的cookie来进行欺骗

3. session会在一定时间内保存在服务器上，但当访问增加，会占用服务器性能

4. 单个cookie保存的数量和长度有限制，最多有20条，而且每条的长度不能超过4KB

所以登录信息等重要的信息应该放在session，其他的保留在cookie

如何删除一个cookie

  1. 将时间点设为当前时间往前一点
    var date = new Date();

    date.setDate(date.getDate() - 1); //真正的删除，setDate()方法用于设置一个月的某一天

  2. expires的设置
    document.cookie = ‘user=’ + encodeURIComponent(‘name’) + ‘;expires = ‘ + new Date(0);

谈谈你对语义化的理解

1. 去掉或者丢失样式时也能够让页面呈现出清晰的结构

2. 有利于SEO和搜索引擎建立良好的沟通，有助于爬虫抓取更多有效的信息（爬虫依赖标签来确定上下文和各个关键字的权重）

3. 方便其他设备解析（如屏幕阅读器、盲人阅读器、移动设备，不同的标签的可能有不同的声音等等）

4. 便于团队开发和维护，语义化更具有可读性，可以把握网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化

Doctype作用，严格模式与混杂模式如何区分

1. <!Doctype>声明位于文档的最前面，告知浏览器以何种模式来渲染文档

2. 严格模式的排版和JS运作模式是以该浏览器支持的最高标准运行

3. 在混杂模式中，页面以宽松的向后兼容的方式显示。模拟老式的浏览器的行为以防止站点无法工作

4. Doctype不存在或者格式不正确会导致文档以混杂模式出现

HTML和XML的区别

1. 所有的标记都必须有一个相应的结束标记

2. 所有的标签元素和属性的名字都必须使用小写

3. 所有的XML标记都必须合理嵌套

4. 所有的属性必须用引号""括起来

5. 把所有的<和&特殊符号用编码表示

6. 给所有的属性赋一个值

7. 不要在注释内容中使用"--"

8. 图片必须要有说明文字

上下margin重合问题

IE和Firefox都存在，相邻的两个div的margin-left和margin-right不会重合，但是margin-top和margin-bottom却会发生重合

解决的办法：养成良好的代码习惯，同时采用margin-top或者同时用margin-bottom

  2. 浏览器默认的margin和padding不同，解决方案：加一个全局的*{margin:0;padding:0;}来统一

浮动

浮动的框可以向左或者向右移动，直到它的外边缘碰到包含框或者另一个浮动框的边框为止。由于浮动框不在标准文档流中，所以文档的普通流的块框就当浮动框不存在。浮动的块框会漂浮在普通文档流之上。

清除浮动的技巧

1. 使用空标签清除浮动
这种方法是在所有的浮动标签后面添加一个空的标签定义css clear:both; 弊端就是增加了无意义的标签

2. 使用overflow
给包含浮动元素的父标签添加CSS属性overflow：auto；zoom:1;  zoom:1;是为了兼容IE6

3. 使用after伪对象清除浮动
该方法只适用于非IE浏览器。具体方法可以参照以下实例。注意height：0；否则该元素会比实际要高几个像素
.clearfix:after{
    content: ".";
    display: block;
    clear: both;
    height: 0;
    overflow: hidden;
    visibility: hidden;
}
.clearfix{
    zoom: 1;
}

4.我常用的,before是为了防止浏览器顶部的空白奔溃
.clearfix:before,.clearfix:after{
    content: " ";
    display: table;  //匿名的表格单元
}
.clearfix:after{
    clear: both;
}

display:none 和 visibility:hidden的区别

display：none 隐藏对应的元素，在文档布局中不再给它分配空间，就当它没有存在过

visibility：hidden 隐藏对应元素，但是在文档中仍然保留原来的空间

CSS中的link和@import的区别

1. link属于HTML标签，而@import是CSS提供的

2. 页面被加载时，link会同时被加载，而@import会等到引用它的CSS文件被加载完再加载

3. @import只在IE5以上才会被识别，而link是HTML标签,没有兼容问题

4. link的权重大于@import

position:absolute和float属性的异同

- 共同点：对内联元素设置float和absolute属性，可以让元素脱离文档流，并可以设置其宽高

- 不同点：float仍然会占据位置，absolute会覆盖文档流的其他元素

介绍一下box-sizing属性

box-sizing属性主要用来控制元素的盒模型的解析模式。默认值是content-box.

- content-box：让元素维持W3C的标准盒模型。元素的宽度/高度由 border+padding+content的宽度/高度决定，设置的宽/高属性指的是content的宽/高

- border-box：让元素维持IE传统盒模型（IE6以下版本和IE6~7的怪异模式）。设置的宽/高属性指的是border+padding+content的宽/高

标准浏览器下，按照W3C规范对盒模型解析，一旦修改了元素的边框和内距，就会影响元素的盒子尺寸。

null和undefined的区别

null是一个表示“无”的对象，转为数值为0；null用来表示尚未存在的对象，常用作表示函数企图返回一个不存在的对象。null表示“没有对象”，即该处不应该有值。典型的用法是：

1. 作为函数的参数，表示该函数的参数不是对象

2. 作为对象原型链的终点

undefined是一个表示“无”的原始值，转化为数值是NaN。当声明还没有初始化的时候，变量的默认值为undefined。它表示“缺少值”。典型的用法：

1. 变量声明了，但还未赋值，等于undefined

2. 调用函数时，应该提供的参数没有提供，该参数等于undefined

3. 对象没有赋值的属性，该属性的值为undefined

4. 函数没有返回值时，默认返回undefined

new操作符

2. 属性和方法被加入到this引用的对象中

3. 新创建的对象由this引用，并且最后隐式返回this

一个页面从输入URL到页面加载显示完成，这个过程中都发生了什么？

分为四个步骤：
1. 当发送URL请求时，不管这个URL是Web页面的URL还是Web页面上每个资源的URL，浏览器都会开启一个线程来处理这个请求，同时在远程DNS服务器上启动一个DNS查询。这能使浏览器获得请求对应的IP地址。

2. 浏览器与远程Web服务器通过TCP三次握手协商来建立一个TCP/IP连接。该握手包括一个同步报文，一个同步-应答报文和一个应答报文。这三个报文在浏览器与服务器间传递。该握手首先由客户端尝试建立通信，而后服务器应答并接受用户的请求，最后由客户端发出该请求已经被接受的报文。

3. 一旦TCP/IP连接成立，浏览器会通过该连接向远程服务器发送HTTP的GET请求。远程服务器找到资源并使用HTTP响应返回该资源

4. 此时，web服务器提供资源服务，客户端开始下载资源

javascript的同源策略

概念：同源策略是客户端脚本（尤其是javascript）的重要的安全度量标准。这里的同源策略指的是：协议，域名，端口相同，是一种安全协议。指一段脚本只能读取来自同一来源的窗口和文档的属性。
